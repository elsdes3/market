Explore Best Model Predictions

About

Overview

As mentioned in the scope, understanding ML predictions of propensity can be helpful to characterize audience groups during the design of a marketing campaign. One example question could be answered by exploring the predictions of a ML model is what are the most and least important attributes of visitors predicted to have a high or low propensity. This can be one factor used to design a marketing strategy aimed at one or both groups of visitors.

This step explores the best ML model’s predictions of visitor propensity to make a purchase on a return visit using ML explainability packages (eg. SHAP). This exploration is done per audience group so the output of this step is a profile of each audience group based on the features that are most important to predicting the outcome for first-time visitors to the store in that group during the inference data period. This is performed as part of step 4. from a typical A/B Testing workflow.

This step can be run prospectively at the end of the inference period, just before the start of the campaign, when all the inference data (first-time visitors to the store) becomes available.

Technical Requirement

Model exploration is supported for a single ML model. This is the best model found in an earlier step and registered in the MLFlow model registry. The model associated with a single experiment run (or run_id) is a sklearn Pipeline, consisting of data preprocessing and ML modeling steps. SHAP cannot work directly with a sklearn Pipelines to generate explanations. It can only accept the ML model (classifier, or last Pipeline step). So, we’ll need to retrieve the preliminary Pipeline steps (the data preprocessor) and the transformed data that is generated by the preprocessor. Then, we can make predictions on this transformed data using the last step (the ML classifier). Finally, SHAP can be used to explain predictions made by this classifier. These explanations are required for the first-time visitors to the store during the inference data period.

Limitations

  1. Since extracting SHAP values can be time-consuming, they will be retrieved for a slice of the inference data and not for the entire inference dataset.

Implementation

First, the preprocessor steps of the Pipeline are used to get the transformed train and test data splits from the best model. A custom Python modeule src/model_explore_helpers.py has been developed to extract the transformed train and data. Next, the ML classifier step of the Pipeline is used to make and score predictions of the processed test data using this model.

Next, the end-to-end Pipeline will be used to make and evaluate these predictions directly on the raw test data, without creating the intermediate transformed data splits.

Then, the metrics using both approaches will be compared to ensure they are identical to each other. This is outlined below

  1. Make and evaluate predictions using the model (end-to-end sklearn Pipeline) associated with the specified run_id
  2. Make and evaluate predictions using the manual approach
  3. Compare predictions and evaluation metrics on the training and test data splits between the Pipeline-based and manual outputs

If the Pipeline and manual approaches are verified to give identical metrics for first-time visitors in the test data split during ML model development, then it is possible to reliably apply the same manual workflow to retrieve the transformed features during the inference period. So, next, this same manual transformation procedure is applied to transform the inference data. Recall that the inference data only covers the first visit during the inference period and so the outcome of the return visit for each first-time visitor is not known until after the entire inference (production) period (one month) has ended.

Finally, a SHAP explainer will be used to explore a subset of the transformed inference data in order to get the feature importances for the audience group(s). The top N most important features for each audience group will be combined into a single DataFrame and exported to disk. This will represent the profile of the audience group(s) in terms of most important features.

User Inputs

Define the following

  1. best MLFlow run ID
  2. name of column containing label (outcome)
  3. list of categorical features
  4. list of numerical features, including categorical features present in the raw data as integers
  5. number of SHAP observations from the transformed inferece data that are to be explained
  6. number of features from the transformed inference data for which SHAP feature importances are required
  7. audience_groups_strategies
    • dictionary of desired audience groups into which the first-time visitors propensities will be placed
      • key specifies the number of desired audience propensity groups (low, medium and high)
      • value specifies names of the desired audience propensity groups
  8. type of audience strategy (single- or multi- group) from which to create cohorts
# 1. 
best_run_id = "742cf152d5094be0b686edf910c7d398"

# 2. label column
label = "made_purchase_on_future_visit"

# 3. categorical columns
categorical_features = [
    'bounces',
    'source',
    'medium',
    'channelGrouping',
    'last_action',
    'browser',
    'os',
    'deviceCategory',
]

# 4. numerical columns
numerical_features = [
    'hits',
    'pageviews',
    'promos_displayed',
    'product_views',
    'product_clicks',
    'time_on_site',
]

# 5. number of SHAP observations to be explained
num_shap_obs = 500

# 6. number of top-performing SHAP features to be kept
num_features_shap = 10

# 7. mapping dictionaries
audience_groups_strategies = {
    1: ["High", "Medium", "Low"], 2: ["High", "High-Medium", "High-Medium-Low"],
}

# 8. type of audience strategy to use when creating groups
audience_strategy = 1

ML Evaluation From Pipeline Using Transformed Data

Get Transformed Data

In order to get the transformed data used by the end-to-end Pipeline, the following two-step procedure will be used

  1. Extract untrained pipeline consisting of the following steps
    • under-sampling
      • under-sampling
      • pre-processing
    • over-sampling
      • pre-processing
      • over-sampling
  2. Use untrained pipeline to get transformed training and validation data as follows
    • under-sampling
      • under-sample training data
      • train pre-processor using under-sampled training data
      • pre-process all
        • training data
        • validation data
    • over-sampling
      • train pre-processor using all training data
      • pre-process all
        • training data
        • validatation data
      • over-sample pre-processed training data
if 'Under' in type(pipe.named_steps['resampler']).__name__:
    resampling_approach = 'us'
    pipe_resample_trans = meh.get_preprocessor_pipeline(pipe, 'us')
    X_train_trans, X_test_trans = meh.get_preprocessed_resampled_data(
        'us',
        X_train,
        y_train,
        X_test,
        categorical_features,
        pipe_resample_trans=pipe_resample_trans,
    )
else:
    resampling_approach = 'os'
    pipe_trans, pipe_resample = meh.get_preprocessor_pipeline(pipe, 'os')
    (
        X_train_trans,
        X_test_trans,
        X_train_trans_rs,

        'os',
        X_train,
        y_train,

        categorical_features,

        pipe_resample=pipe_resample,

Explore Model

For each audience group, profile the audience group(s) using SHAP feature importances as follows

  1. get a random slice of the inference data within each audience group
  2. get SHAP values for the selected slice of inference data
  3. append metadata about the selected slice of inference data

The above process is repeated for all audience groups. This is shown below

dfs_shap_values = []
for maudience in audience_groups_strategies[audience_strategy]:
    # 1. Get SHAP training data for audience group
    df_infer_shap_train = df_infer_trans.sample(n=num_shap_obs)
    print(
        f"Randomly sliced {num_shap_obs:,} observations "
        f"from {maudience} audience group in inference data"
    )

    # 2. Get SHAP values
    start_time = datetime.now(pytz.timezone("US/Eastern"))
    start_time_str = start_time.strftime("%Y-%m-%d %H:%M:%S.%f")
    print(f"Query execution start time = {start_time_str[:-3]}...", end="")
    # shap_means = np.abs(shap_values).mean(axis=0)
    shap_means = np.random.rand(num_features_shap, 1)
    end_time = datetime.now(pytz.timezone("US/Eastern"))
    end_time_str = end_time.strftime("%Y-%m-%d %H:%M:%S.%f")
    duration = end_time - start_time
    duration = duration.seconds + (duration.microseconds / 1_000_000)
    print(f"done at {end_time_str[:-3]} ({duration:.3f} seconds).")
    print(
        f"Retrieved SHAP values using {num_shap_obs:,} observations "
        f"from {maudience} group in inference data"
    )

    # 3. Log SHAP values and metadata
    df_shap_values = (
        pd.DataFrame.from_dict(
            {
                'feature': list(X_trans_rs)[:num_features_shap],
                'mean_shap_value': shap_means.squeeze(),
            },
            orient='index'
        ).transpose()
        .sort_values(by='mean_shap_value', ascending=False)
        .reset_index(drop=True)
        .assign(infer_month=month_name[X_infer_with_audience['month'].squeeze()[0]])
        .assign(maudience=maudience)
        .assign(audience_strategy=audience_strategy)
        .assign(num_shap_observations=num_shap_obs)
    )
    dfs_shap_values.append(df_shap_values)
